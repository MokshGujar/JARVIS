# Repository Cleanup Audit

Date: 2026-05-16
Branch: `chore/repo-cleanup-audit`

## Purpose

This cleanup removes repository clutter without changing application behavior. It is limited to generated files, build output, local caches, Git tracking of runtime/local state, ignore rules, and cleanup documentation.

No frontend/HUD behavior, Android source behavior, `AutomationService` behavior, Face Gate, speaker verification, step-up authentication, or tool execution policy behavior was changed.

## SAFE_TO_DELETE

Deleted where filesystem permissions allowed because these are generated and reproducible:

- Python bytecode/cache folders: root `__pycache__/`, `app/**/__pycache__/`, `tests/__pycache__/`.
- Test runtime scratch data: `tests/_tmp/`.
- Android build output: `android-companion/app/build/`.
- Android Gradle generated cache folder: `android-companion/.gradle/`.

Generated folders that remain because the local filesystem denied deletion:

- `.pytest_cache/`
- `android-companion/build/`

## REMOVE_FROM_GIT_TRACKING_ONLY

These are used locally or generated by local tools, so they should remain available on disk when present but should not be committed:

- `database/learning_data/userdata.txt`
- `database/observability/recent_events.json`
- `database/permissions/pending_approvals.json`
- `database/phone_bridge/devices.json`
- `database/phone_bridge/pending_actions.json`
- `database/reminders/reminders.json`
- `android-companion/.gradle/**`
- `android-companion/local.properties`

## ADD_TO_GITIGNORE

The root `.gitignore` now covers:

- Python caches, bytecode, virtual environments.
- Environment files, local config, credentials, token files, PEM/key material.
- Logs and process output.
- Runtime databases and local `database/` state.
- Android local Gradle state, IDE state, and `local.properties`.
- Temp files and `tests/_tmp/`.
- IDE folders.
- Node/frontend build output.
- Tooling caches and coverage output.
- ML/model files.
- Generated captures and media.
- OS metadata files.

`frontend/audio/*.mp3` is explicitly unignored because the repository already contains tracked starter audio assets that are source assets, not disposable generated media.

## MUST_KEEP

These areas were intentionally preserved:

- `app/` source code, including automation, auth, policy, Face Gate, speaker/voice-related services, tools, and orchestrator code.
- `tests/` source tests.
- `frontend/` source, HUD assets, launcher/enroll pages, and tracked starter audio.
- `android-companion/app/src/`, Gradle wrapper, Gradle build scripts, manifest, resources, and Android source files.
- `config/`, `config.py`, `requirements.txt`, `requirements-dev.txt`, `run.py`, and `DEVELOPMENT.md`.
- `docs/` and existing tracked `reports/` files.
- Local `.env`, `.venv/`, `.vscode/`, and Android IDE state were not deleted because they are user-local files.
- Local runtime state under `database/` was preserved on disk.
- Local Android `speaker_verify.onnx` was preserved on disk because Android speaker verification explicitly references it.

## REVIEW_MANUALLY

These items may be historical, generated, obsolete, or personal, but were not deleted because removal could lose useful context or affect local behavior:

- `reports/`, including historical phase reports and `reports/openjarvis_vs_jarvis_full_comparison.md`.
- Local `database/agent_tasks/`, `database/chats_data/`, `database/camera_captures/`, `database/face_identity/`, `database/memory/`, `database/vector_store/`, `database/voice_identity/`, and `database/runtime_state.sqlite3`.
- Local `.env`.
- Local `.vscode/` and `android-companion/.idea/`.
- Local `android-companion/app/src/main/assets/speaker_verify.onnx`.
- Any untracked local runtime files not explicitly listed as generated cache/build output.

## Runtime Data Policy

Runtime folders are ignored because this app stores local assistant state, chat history, reminders, phone bridge state, vector indexes, camera captures, observability events, pending approvals, and identity material under the repository tree. These files are machine-local and may contain personal data or biometric/security-sensitive state.

Local identity, vector, database, and phone bridge files should not be tracked because committing them can leak private data, create noisy diffs, and make behavior depend on one developer machine's runtime state.

## Risks And Assumptions

- Assumption: Python bytecode, pytest cache, test scratch files, and Android build outputs are reproducible.
- Assumption: `database/` content is runtime/local state and must be preserved locally but removed from Git tracking.
- Risk: A fresh clone will not include local runtime state; the app must recreate state or the user must restore private local data separately.
- Risk: `android-companion/app/src/main/assets/speaker_verify.onnx` is ignored as a model file. It must be supplied locally for Android speaker verification.
- Risk: Some `reports/` files may be obsolete, but they were kept because they may document previous decisions and tests reference at least some report paths.
- Risk: `.pytest_cache/` and `android-companion/build/` remained locally due filesystem permission errors. They are ignored and not tracked.

## Rollback And Recovery

To restore Git tracking changes before committing:

```powershell
git restore --staged .
git restore .gitignore docs/REPO_CLEANUP.md
```

To restore removed generated files, rerun the relevant tool:

```powershell
python -m compileall app
python -m pytest
.\android-companion\gradlew.bat assembleDebug
```

To recover untracked local runtime data after accidental local deletion, restore it from a private backup. Runtime data intentionally should not be recovered from Git once removed from tracking.
